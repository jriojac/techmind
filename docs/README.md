# 📚 Documentación de TechMind

Bienvenido a la documentación técnica de **TechMind**.

Esta carpeta reúne todos los documentos relacionados con la arquitectura, diseño, planificación y estándares del proyecto.

---

# Estructura

| Carpeta | Descripción |
|----------|-------------|
| ADR | Registros de decisiones de arquitectura (Architecture Decision Records). |
| Architecture | Diagramas y documentación de la arquitectura del sistema. |
| Roadmap | Planificación del proyecto, hitos y evolución. |
| SDS | Software Design Specification. |
| Standards | Estándares de desarrollo, documentación y buenas prácticas. |
| MeetingMinutes | Actas y minutas de las reuniones del equipo. |

---

# Guía de Navegación

## Arquitectura

Contiene los diagramas y documentos que describen la estructura del sistema.

```
docs/Architecture/
```

---

## Software Design Specification (SDS)

Describe el diseño funcional y técnico del proyecto.

```
docs/SDS/
```

---

## Architecture Decision Records (ADR)

Documenta las decisiones arquitectónicas importantes tomadas durante el desarrollo.

```
docs/ADR/
```

---

## Roadmap

Describe la planificación del proyecto, los sprints y la evolución prevista.

```
docs/Roadmap/
```

---

## Standards

Incluye las normas utilizadas por el equipo para garantizar consistencia en el desarrollo y la documentación.

Ejemplos:

- Convenciones de código.
- Estándares para diagramas Mermaid.
- Convenciones para documentación.
- Flujo de trabajo con Git.

```
docs/Standards/
```

---

## Meeting Minutes

Contiene las actas de reuniones técnicas, revisiones de sprint y acuerdos del equipo.

```
docs/MeetingMinutes/
```

---

# Organización de la Documentación

La documentación de TechMind se organiza en cuatro niveles:

1. **README.md (raíz del proyecto)**  
   Presentación general del proyecto.

2. **docs/README.md**  
   Índice de la documentación técnica.

3. **README.md de cada componente**  
   Documentación específica de Backend y Ciencia de Datos.

4. **Documentos especializados**  
   SDS, ADR, Roadmap, Standards y Meeting Minutes.

---

# Mantenimiento

Toda nueva documentación deberá incorporarse en la carpeta correspondiente y mantenerse actualizada conforme evolucione el proyecto.
